# Aspector

## 基本概念

1. 切面(Aspect): 切面是描述程序如何对类型进行**横向分析**与**附加行为**的模板，Aspector中的切面是由class定义的，一个切面就是一个class，其形式与java类型的声明无异
2. 切面扩展(Aspect Extends): 一个切面可以扩展自若干个其他切面，可以构成一个独立于java类型继承的层次结构，并且允许扩展自多个其他切面。
3. 切面元素(Aspect Element): 在描述切面的类型中定义的任何**非私有非final的成员方法**均视为切面元素，所有的切面元素将按照其通过`@AspectElement`描述的**切入方法**参与切面声明的构造，未携带此注解的元素切入方法将视为`OVERRIDE`。
4. 切入方法(Aspect Using): 决定切面元素参与切面声明构造的方式，默认为OVERRIDE。
5. 切面声明(Aspect Declare): 描述若干切面应用至一个被切入目标类的结构对象，它包含了若干方法，可获取此切面声明所构造的已分析和附加行为后的类型数据，包括其类名与原始二进制字节码等。
6. 共享字段(Shared Field): 在切面中声明的所有字段均默认视为孤立于所有其他切面，即字段仅在其声明的切面内部生效，但是当使用`@Shared`注解将字段标记为共享后，字段将在一个切面声明内的所有层次中的**同名共享字段**内共享并传递数据。
7. 存根(Stub): 切面并不明确其应用到的被切入类型声明，但是当可以明确切面的可作用类型范围，甚至是可明确可被应用的目标类型包含的成员范围时，可通过java语义下的类型继承将此类型扩展/实现到一个切面的class，并将之标记为`Stub`存根，之后在切面声明中对存根的访问将被重定向至存根附近到的目标，默认情况下为被应用到的被切入目标类型。
